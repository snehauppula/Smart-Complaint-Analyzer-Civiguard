Civic Complaint Analyzer

Classify citizen complaints and surface urgency — in one flow.

🎯 Problem statement

  • Municipal teams receive mixed free-text complaints (roads, water, power, waste).
  • Manual triage is slow and inconsistent; urgent issues can be buried.
  • Need: automatic category routing plus a transparent priority hint for responders.

✨ Solution overview

  • ML classifier predicts road · water · electricity · garbage from complaint text.
  • Confidence score shows how sure the model is (0–100%).
  • Rule-based priority (LOW / MEDIUM / HIGH) runs on raw text, with short explainability (why that band).
  • Streamlit UI for demos, interviews, and quick revision.

  • Preprocessing: normalize text, custom stopword handling, lemmatization (keeps meaning, reduces noise).
  • Output: category + confidence + priority band + score + human-readable reasons.

🛠️ Technologies used

  • Python — core language
  • NLP — tokenization, POS-aware lemmatization (WordNet)
  • TF-IDF — text → numeric vectors (unigrams + bigrams)
  • Multinomial Naive Bayes — fast probabilistic classifier
  • scikit-learn pipeline — vectorizer + model in one object (train / save / load)
  • Streamlit — interactive web UI
  • Rule-based layer — keywords, phrases, and duration patterns for priority

📊 Model (simple)

  • Why TF-IDF? Strong baseline for short civic text; highlights discriminative words/phrases; works well with sparse bag-of-n-grams.
  • Why Naive Bayes? Fast, interpretable, strong with count-based features; good for multiclass text when data is modest.
  • Confidence score: max predicted class probability from the model — higher means the model is more decisive for that input.

⭐ Key features

🏷️ Classification

Four civic categories for routing and analytics.

📈 Confidence

Percent score to flag uncertain cases for review.

🚨 Priority

LOW / MEDIUM / HIGH + numeric score (rule-based, not from NB).

💡 Explainability

Short bullets: which rules fired (safety, duration, leakage, etc.).

⚠️ Challenges faced

  • Dataset quality & size — realistic, balanced civic phrases take effort.
  • Misclassification — overlapping words across domains (e.g. “near market”, “pipeline”).
  • Low confidence — vague or very short complaints.
  • Inference vs training — predictions must use the same cleaning as training.

✅ How they were addressed

  • Expanded / cleaned dataset — more diverse wording per category.
  • Consistent preprocessing for train and inference (fixes many “wrong bucket” cases).
  • TF-IDF n-grams — captures short phrases, not only single words.
  • Train/test split before vectorization — avoids leakage and gives honest metrics.
  • Rule-based priority — separate layer for safety and duration signals ML might not label.

🏁 Results

~94% test / CV accuracy (approx.)

  • Strong performance on clear, domain-specific complaints.
  • Confusion mostly on ambiguous or multi-topic sentences — use confidence as a guardrail.
  • Model serialized as a single pipeline file for reproducible deployment.

🚀 Future improvements

  • BERT / transformers — better semantics for noisy or informal text.
  • Supervised priority model — learn from labeled urgency instead of rules only.
  • Multilingual support and active learning from operator corrections.
  • API + ticketing integration for real-time civic workflows.
